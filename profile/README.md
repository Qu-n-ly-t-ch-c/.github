How To Configure FirewallD on RHEL 7 or CentOS 7 | VCP Blog
The command firewall-cmd --runtime-to-permanent saves the current, active firewall rules to the permanent configuration, so they will persist after a service reload or reboot. This is used after making changes to the runtime configuration, which is often done for testing purposes. 
How it works
Runtime vs. Permanent: firewalld has two separate configurations: the "runtime" configuration, which is the live, active state, and the "permanent" configuration, which is stored in configuration files.
Default behavior: By default, firewall-cmd commands only change the runtime configuration. These changes are lost when firewalld is reloaded or the system restarts.
Making changes permanent: To make changes permanent, you must either use the --permanent option when making the change (which requires a separate --reload command) or use the --runtime-to-permanent command to save the current runtime settings. 
When to use it
Use this command after you have made a series of temporary changes to the firewall rules at runtime and are satisfied with the results.
It is a good way to test a new set of rules before committing them permanently. 
Example
Add a service to the runtime configuration:
sudo firewall-cmd --add-service=ssh
Make the change permanent:
sudo firewall-cmd --runtime-to-permanent
